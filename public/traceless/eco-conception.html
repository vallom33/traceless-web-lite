Principes d'éco-conception web

L'éco-conception web vise à réduire l'impact environnemental des sites et applications tout en améliorant l'expérience utilisateur. Voici les principes clés :

1. Réduire le poids des médias

Les images, vidéos et polices représentent souvent 80% du poids d'une page. Pour réduire leur impact :

  • Compresser les images (WebP, AVIF, SVG)
  • Utiliser des dimensions adaptées (pas de 4K pour une miniature)
  • Charger les médias uniquement à la demande (lazy loading explicite)
  • Éviter les vidéos en autoplay
  • Préférer le texte aux images décoratives

EN: Compress images, use appropriate dimensions, load media only on demand.

2. Prioriser le texte

Le texte est le format le plus léger et le plus accessible. Un paragraphe de 1000 caractères pèse environ 1 Ko. La même information en image peut peser 100 Ko.

  • Utiliser du texte pour le contenu principal
  • Réserver les images pour l'information qui ne peut pas être textuelle
  • Fournir des alternatives textuelles (attribut alt)
  • Structurer avec du HTML sémantique

EN: Text is the lightest and most accessible format. Use semantic HTML.

3. Limiter JavaScript et dépendances

JavaScript moderne peut facilement ajouter des centaines de Ko à une page. Chaque bibliothèque importée a un coût :

  • React + ReactDOM : ~140 Ko (gzippé)
  • jQuery : ~30 Ko (gzippé)
  • Moment.js : ~70 Ko (gzippé)
  • Ce site : ~1 Ko de JS (optionnel)

Préférez le JavaScript natif, ou mieux : pas de JavaScript du tout quand c'est possible. HTML et CSS peuvent faire beaucoup.

EN: Avoid heavy frameworks. Native JS or no JS is often sufficient.

4. Limiter pistage et cookies

Les scripts de pistage (Google Analytics, Facebook Pixel, etc.) ajoutent :

  • Du poids supplémentaire (50-200 Ko par tracker)
  • Des requêtes HTTP additionnelles
  • Une atteinte à la vie privée des utilisateurs
  • Une dette technique (maintenance des consentements)

Alternative : logs serveur anonymisés, ou simplement aucune mesure d'audience. Avez-vous vraiment besoin de savoir combien de visiteurs vous avez ?

EN: Trackers add weight and privacy concerns. Consider not tracking at all.

5. Concevoir pour tous les utilisateurs

Un site éco-conçu est naturellement plus accessible :

  • Connexions lentes (3G, zones rurales)
  • Appareils anciens ou peu puissants
  • Utilisateurs de lecteurs d'écran
  • Navigation au clavier
  • Navigateurs texte (w3m, lynx, links)

Le web sobre est un web inclusif.

EN: A low-weight web is an inclusive web. Design for all users.
